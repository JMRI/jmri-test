JMRI: Utilisation de JavaDoc et UML

la bibliothèque JMRI fournit la majorité de ses Documentation de l'API via JavaDoc et UML .

Nous utilisons également les annotations Java pour documenter la structure de notre code.

Nous utilisons les UMLGraph doclet pour créer automatiquement les diagrammes UML dans nos pages JavaDoc.

JavaDoc pour les Développeurs

Le tutoriel JavaDoc est une bonne chose, mais un peu long, introduction pour JavaDoc .

UML et UmlGraph pour les Développeurs

UML est un langage large et profond pour décrire la structure, la conception et le comportement des systèmes informatiques. JMRI utilise principalement les diagrammes de classes UML pour documenter sa structure. De nombreux tutoriels UML sont disponibles sur le web. En particulier, un bon point de départ est l' introduction à la classe diagrammes Object Mentor . Pour plus de détails, s'il vous plaît voir cours Atlas UML lien cassé au 14/01/2012 ou introduction Embarcadero Software .

Processus

Le norme JMRI Ant fournir trois objectifs documentation:
javadoc -Créer des JavaDocs texte dans la doc local/. Doc Ouvrir/index.html pour l'accès.
javadoc-uml -Créer des diagrammes UML et JavaDocs dans la doc local/. Doc Ouvrir/index.html pour l'accès.
uploadjavadoc -Envoyer la documentation actuelle dans la doc local/sur le site web jmri.org.
