JMRI: Construire avec NetBeans

NetBeans 6.8 (disponible sur le site www.netbeans.org ) est une plate-forme idéale pour travailler avec JMRI. Le code JMRI est configuré pour travailler avec NetBeans avec presque pas de configuration supplémentaire. (La description ci-dessous est exacte pour la version Mac OS X de NetBeans 6.8. NetBeans est censé être identique sur toutes les plateformes, mais cela n'a pas été vérifiée pour toutes). NetBeans est un environnement très puissant de développement intégré (IDE), qui peut rendre le travail avec le code JMRI assez facile. Mais cela prend un peu de temps pour s'y habituer, et en lire les tutoriels sur le site web depuis NetBeans 6.8 sera utile.

Obtenir et installer Netbeans

  • Télécharger Netbeans 6.8 (La version Java SE est très bien) de netbeans.org
  • Exécutez le package d'installation de netbeans

Vérification des codes

La meilleure façon d'obtenir la copie initiale du code est de le vérifier directement à l'aide du CVS intégré de Netbean. Une fois que vous avez fait cela, NetBeans l'utilisera automatiquement.

Démarrer NetBeans et suivre les étapes suivantes. Le texte entre crochets [] sont des éléments de menu à choisir ou des boutons à presser. Le texte dans cette police doit être tapé dans NetBeans exactement comme écrit. Cette police désigne des explications et des indications scéniques à suivre.

[Team]->[CVS]->[Checkout]
  • CVS Root: : pserver: anonymous@jmri.cvs.sourceforge.net: /cvsroot/jmri
  • Mot de passe: laisser vide
  • [Next]
  • Module: all (Nous avons vraiment l'intention de saisir «tous» dans le champs Ne pas laissez le champ vide.!)
  • Branch: laisser vide
  • Local Folder (dossier loca)l: prend un peu de place sur votre> disque dur ...
  • [Finish]

Générer et exécuter JMRI

Pour générer et exécuter DecoderPro, tout ce que vous avez à faire est de sélectionner "Run Project (JMRI)" dans le menu Run, ou cliquez sur l'icône Run dans la barre d'outils.

  • Run -> Run Project (JMRI) compilera et invoquer DecoderPro
  • Vous pouvez choisir d'autres cibles (par exemple PanelPro ...) en changeant [File] -> [Propriétés] -> [Build and Run] -> [Run Project]

    Si vous souhaitez exécuter le programme sous le débogueur NetBeans, sélectionnez "Debug Project (JMRI)" de la commande "Run" du menu..

    Mettre à Jour le code depuis CVS

    De temps en temps, vous pouvez mettre à jour le code pour correspondre à l'actuel contenu du référentiel. Pour ce faire, utilisez le choix de menu [Team] -> [Update].

    Faire un Fichier Jar

    La plupart du code dans une installation normale de JMRI est dans un fichier appelé jmri.jar. Si vous souhaitez le remplacer par votre version mise à jour, vous devez créer un nouveau fichier jmri.jar à partir de votre code modifié.

    Pour ce faire, ctrl-clic (ou clic droit) sur le nom de "JMRI" dans la fenêtre projet "Projects" . Cela ouvrira un menu contextuel, sur lequel vous trouver "Make Jar File". Sélectionnez cela, et un nouveau fichier jmri.jar sera produits dans le répertoire principal du projet.

    Faire un Fichier Patch (Correctif)

    Un «fichier patch» est un moyen facile de rassembler tous vos changements, même s'ils couvrent plusieurs fichiers, en un seul fichier que vous pouvez ensuite nous envoyer. Il est également facile de le fusionner avec le reste directement directement u code, alors nous préférons grandement que vous l'utilisiez directement .

    Pour créer le fichier:

    • Dans le menu [Team], sélectionnez "Export Diff Patch ..."
    • Un navigateur de fichiers s'ouvre, entrez un nouveau nom de fichier à l'endroit de votre choix et cliquez sur "Save" ou "Open", selon ce que montre le dialogue.
    • Après il termine le calcul de la diffs, le fichier que vous avez sélectionné contiendra les patchs, avec un bandeau d'information.

    Détails Windows

    Lors de l'exécution à partir de Netbeans sur un système Windows, si JMRI ne trouve pas 
votre connexion réseau c'est peut-être le problème.

Le problème est dû à un autre environnement d'exécution Java (JRE) qui est
utilisé lorsqu'il est exécuté à partir de Netbeans, par opposition à un normal.

Normalement, l'installateur Windows offre la copie de la bibliothèque
 de fichiers de communications dans le JRE - Cependant, vous devrez le faire
manuellement dans ce cas.

1. from C:\Program Files\Java\jre1.7.0\lib\ext < Serialio.jar>
into 'C:\Program Files\Java\jdk1.7.0\jre\lib\ext'

2. from C:\Program Files\Java\jre1.7.0\bin  and 
into 'C:\Program Files\Java\jdk1.7.0\jre\bin'
