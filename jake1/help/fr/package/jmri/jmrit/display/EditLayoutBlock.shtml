Créer/Modifier UnCanton

Les cantons sont des tronçons de voie dont l'occupation peut être suivie individuellement. Diviser la voie en cantons est facultative. Les cantons mis en place dans un schéma de voies correspondent normalement à des cantons physiques sur un réseau. L'occupation du canton est indiqué par l'état d'un capteur d'occupation. Lorsque le tracé d'un réseau est divisé en cantons, toutes les voie ne doivent pas appartenir à un même canton. Par exemple la voie de triage, n'est normalement pas dans un canton, alors que la voie principale sera toujours divisée en cantons. La principale utilisation des cantons est de faciliter la signalisation, mais les cantons peuvent être utilisés pour contrôler sur le réseau d'autres fonctions d'animations.

Si un nom de canton est entré dans le champ Canton: Nom de la barre d'outil avant l'ajout d'un aiguillage, d'un croisement, ou un segment de voie, l'élément ajouté est attribué à ce canton, si un nom de capteur (le nom système ou nom utilisateur) a été entré dans le Capteur d'Occupation sur le terrain, ce capteur sera attribué au canton d'entré. Un aiguillage, passage à niveau, ou un segment de voie peuvent également être affectés à un canton en entrant le nom de ce canton dans le dialogue d'édition accessible en sélectionnant Modifier ... dans les éléments du menu contextuel. Dans les deux cas, si un canton avec le nom entré n'existe pas, il est créé.

Un nom de canton peut être n'importe quelle série de caractères alphabétiques ou numériques qui décrivent le canton, par exemple, "Red Main 2", "Lake Siding", ou "Moose Block». Une fois qu'un canton est créé, son nom ne peut être changé; tenter de changer un nom de canton va créer un nouveau canton ayant le nouveau nom. Après tous les éléments affectés à l'ancien canton sont modifiés pour le nouveau canton, le canton avec l'ancien nom sera automatiquement supprimé (voir ci-dessous).

Les cantons peuvent être consultés dans la table canton , qui peuvent être accessibles en utilisant le menu Outils. Les information des cantons pour un Editeur de tracé sont conservée dans deux endroits, qui sont toujours reliés entre eux automatiquement. Pour chaque canton, les information spécifiques de l'Éditeur de Tracés sont enregistrées et gérées dans une partie spécifique de l'Éditeur de Tracé et la partie de l'information du canton est enregistrée dans JMRI Block Quand un canton est créé avec l'Editeur de Tracé , un canton JMRI correspondant est automatiquement créé. Le Canton créé automatiquement aura un nom système IBnnn (où nnn est un nombre), et un nom d'utilisateur le même que le nom du canton mentionné ci-dessus. Ce sont les Cantons génériques qui sont affichés dans la Table Canton. Évitez de changer le nom utilisateur dans la Table Canton, parce que cela va rompre le lien entre les deux parties d'informations du canton .

La partie intéressante dans la Table Cantons est la colonne "Valeur". La «valeur» peut être utilisée pour identifier les trains qui se déplacent autour du tracé. Si un train est garé dans un Canton, et le texte d'identification du train est entré manuellement dans l'entrée "valeur" pour ce Canton, le texte sera propagé de Canton en Canton en suivant le parcours du train sur le réseau . Lorsque le train entre dans un Canton, le texte "Valeur" du canton depuis lequel vient le train est copié dans la "valeur" du Canton dans lequel le train pénètre, quand le train quitte un Canton, la "Valeur" du canton maintenant vide sera effacée Si une variable en mémoire est assignée au canton (voir ci-dessous), la «valeur» de la variable en mémoire sera mise à la "Valeur" du canton. Ainsi en plaçant une icône de la mémoire près du canton, l'identité du train dans le canton peut être affichée dans le panneau de l'Éditeur de Circuit. Attention: Le suivi des trains utilisant cette méthode nécessite d'être bien rodé dans les circuits de détection de canton . De faux contacts ou une détection de canton aléatoire peut perturber le système.

Les aiguillages, les croisements, et les segment de voies éditent des dialogues qui permettent d'accéder au dialogue Créer/Modifier un canton où les informations spécifiques à un canton peuvent être inscrites ou modifiées. Le champs Capteur d'Occupation : dans le dialogue Créer/Modifier du canton indique le nom du capteur d'occupation actuellement affecté au canton, s'il y a un. Pour entrer ou changer le capteur d'occupation, entrez le nom (nom système ou nom utilisateur) d'un capteur dans la Table Capteur dans le champ Capteur d'Occupation:. Un capteur peut être affecté en tant que capteur d'occupation pour un seul canton; tenter de faire autrement se traduira par un message d'erreur. Normalement, l'état d'un capteur d'occupation est "actif" quand un canton est occupé, ainsi "Actif" pour l'occupation est la valeur par défaut automatique. Elle peut être modifiée en sélectionnant «Inactif» dans la boîte de sélection Valeur occupation:.

Les couleurs de voie des cantons sont utilisées au lieu de la couleur de la voie par défaut si une section de voie est dans un canton. Les éléments de voies dans un canton sont dessinés avec différentes couleurs pour les états occupés et inoccupés, permettant à la voie dans un canton de changer de couleur quand un train entre et sort de ce canton. Pour définir les couleurs de voie, sélectionner la couleur de la voie inoccupée dans la boîte de sélection Couleur de Voie: , et sélectionnez la couleur de voie occupée dans la boîte de sélection Couleur de Voie Occupée : . Une autre couleur de la voie inoccupée est prévue pour des utilisations particulières. Par exemple, le Dispatcheur a une option pour utiliser la couleur de voie alternative inoccupées pour les cantons qui sont alloués, mais inoccupée. Réglez la couleur de la voie alternative inoccupée en utilisant la boîte de sélection Couleur de Voie Alternative:. Quand la couleur voie est déterminée l'Éditeur de Circuit utilise la couleur de la voie occupée si un canton est occupé, puis s'il n'est pas occupé, va utiliser la couleur de voie alternative inoccupée si elle a été demandé, sinon il utilise la couleur de voie inoccupée.

Le champs Nom Variable Mémoire: dans la boîte de dialogue du canton Créer/Modifier indique le nom de la variable mémoire actuellement affectée au canton, s'il y a est un. Pour saisir ou modifier la variable mémoire, entrez le nom (nom système ou nom utilisateur) d'une variable mémoire dans la table mémoire dans le champs Nom de Variable Mémoire :. L'affectation d'une variable en mémoire est facultative. Une variable mémoire attribuée contiendra comme «valeur» la «valeur» du Canton, et est utile dans l'identification des trains quand ils sont suivis sur le réseau (voir ci-dessus).

Puisque que les cantons sont des éléments du circuit physique, le même canton peuvent être utilisé dans plus d'un panneau. Par conséquent, l'information canton est sauvegardé dans la partie non spécifique de l'information mise en page du panneau, lorsque les panneaux sont enregistrés dans un fichier sur le disque.

Les cantons sont automatiquement supprimés quand ils ne sont plus utilisés. L'Éditeur de tracé garde la trace du nombre d'éléments de voie (aiguillages, des segments de voie, et des bretelles) qui sont affectés à un canton. Ce nombre est affiché dans le dialogue Créer/Modifier Canton comme Compteur Utilisation Courante:. Lorsque les panneaux sont enregistrés sur le disque, seuls les cantons dont le compteur d'utilisation est supérieur à 0 sont sauvés.
