Guide Utilisateur DecoderPro - Débogage

Pendant que le programme s'exécute

Pendant que le programme s'exécute, l'information de débogage est imprimée sur une "sortie standard", qui est à différent endroits suivant le système d'exploitation que vous utilisez et comment vous démarrez le programme. Sur un Macintosh classique, Elle apparait dans une fenêtre qui est créée avec le premier message d'erreur. Avec Mac OS X, elle peut apparaitre dans une nouvelle fenêtre, ou vous pouvez devoir lancer l'application "Console" pour la voir. Sur Windows, Vous devez exécuter des fichiers batch "Debug" pour voir les messages d'erreur; regardez les dans le répertoire applications.

L'erreur de connexion est contrôler par le fichier"default.lcf". Il y a un lots de commentaires sur ce que signifient les termes variés. Si vous n'aimez pas les messages reliés à un fichier, changer la ligne.

log4j.rootCategory= WARN, A1

Pour inclure la destination enregistrement supplémentaire "R":

log4j.rootCategory= WARN, A1,R

R est défini plus tard dans default.lcf comme connexion pour le fichier "example.log" . Vous pouvez aussi obtenir beaucoup d'informations additionnelles sur ce que fait le programme en réglant le niveau de connexion soit à INFO ou DEBUG par exemple.

log4j.rootCategory= INFO, A1

Notez que le niveau DEBUG sortira beaucoup d'informations,et ralentira significativement l'exécution du programme. Il est possible aussi de requérir plus d'informations juste sur des parties du programme, mais dans l'optique que cela vous soit utile vous devrez passer du temps à apprendre au sujet des parties internes.

Débogage des problèmes avec les fichiers XML

Quand un fichier XML ne peut pas être analyser, vous obtiendrez des messages d'erreur difficiles à comprendre. La partie la plus importante de ceci est le numéro de ligne. Utilisez le pour commencer à voir le problème.

Vous pourriez être capable d'obtenir de meilleures informations en essayant d'ouvrir le fichier avec Internet Explorer. Les informations de débogage qu'il fournit sont généralement plus détaillées que celles fournit par les routines XML dans les librairies JDOM utilisées par JMRI et DecoderPRo.

Notez que Explorer à besoin de voir le fichier .xml et en plus des informations de définitions dans le fichier .dtd. Si vous obtenez un message comme "error in system resource", vous avez besoin de mettre une copie du fichier .dtd (exemple decoder-config.dtd) dans le répertoire avec votre nouveau fichier. DTD peut être trouvé dans le sous-répertoire dtd du répertoire xml.
