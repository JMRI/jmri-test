JMRI Install Guide: Uhlenbrock Intellibox and LocoNet

Introduction

This paper deals with interfacing an Uhlenbrock Intellibox to Digitrax Loconet devices

Background

The Uhlenbrock Intellibox is a multi-protocol digital train control CPU that controls most popular train control protocols including Marklin Digital and the NMRA’s DCC Protocol. It is also implements a compatible version of Loconet developed by Digitrax (www.digitrax.com). I wanted to use a BDL 162 to extend track occupancy detection on my current layout and I encountered some problems with interfacing the BDL 162 to the Intellibox notably, the BDL 162 would not indicate any track occupancy. The BDL 162 acted if there were no power to the layout. No matter what setting was given to the BDL 162, the unit continued indicate that there was no power to the track when in fact there was power to the track.

Solution

After extensive testing, I did not realize that the Intellibox has two different loconet outputs. One for Uhlenbrock throttle devices (IB-Control, FRED) designated Loconet-T. There is another Loconet output designated Loconet-B. I had been using the Loconet-T output to power my Loconet network. I did not realize this. Once I powered the network with the Loconet-B output, the BDL 162 worked properly.

The Loconet-T output can drive more current, so you should probably still use it for Throttles and other LocoNet accessories. Just the BDL162 needs to be connected to the Loconet-B connector.
