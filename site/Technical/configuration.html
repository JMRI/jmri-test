This page discusses the Spring 2002 effort to add configuration persistance to JMRI.

See also the base page for technical documentation.

Introduction

We want to be able to:
  • Save and restore program configuration information
  • Have the stored info be in a human readable format so that people can check that it's correct
  • Have reasonable possibility for manual edits of the configuration info
  • Keep the effort to write configuration code reasonable
  • Preserve some possibility of changing the storage method and format

XML seems the clear choice for readability, edits, and future flexibility. We previously used XML for the storage of decoder and programmer definitions, and that went OK. We'll therefore build on that, changing as needed.
