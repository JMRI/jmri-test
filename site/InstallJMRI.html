Installing the JMRI demos

Download the zipped version of the WIRED client code (about 7M) to a fresh directory, such as c:\WiredBaBar.
  • Through the web, you can find the file at the following location (make sure the save dialog maintains the file extension as .zip, some versions of NT have been seen to set this to .exe, you should force it back to .zip):
    • For JDK1.1: http://www.slac.stanford.edu/~perl/babarwired/jdk1.1/BaBarClient.zip
    • For JDK1.2: http://www.slac.stanford.edu/~perl/babarwired/jdk1.2/BaBarClient.zip
    • For JDK1.3: http://www.slac.stanford.edu/~perl/babarwired/jdk1.3/BaBarClient.zip
  • Through afs, you can find the file at
    • For JDK1.1: /afs/slac.stanford.edu/u/ey/perl/public_html/babarwired/jdk1.1/BaBarClient.zip
    • For JDK1.1: /afs/slac.stanford.edu/u/ey/perl/public_html/babarwired/jdk1.2/BaBarClient.zip
    • For JDK1.3: /afs/slac.stanford.edu/u/ey/perl/public_html/babarwired/jdk1.3/BaBarClient.zip

Unzip the file into that same directory.

  • Special for Macintosh:
    • Unpack the WiredBaBarForMac_sea.bin file with your favorite utility. A self-extracting archive will create a "Wired for Macintosh" folder. The README file in that directory will tell you how to complete the install.
    • Note that for now only the CES_ANALYSIS and CES_BETA startup procedures will work as advertised, but once these applications are started you can change to any other server through the event chooser.
  • Special for NT:
    • Set the environment variable "WIRED_HOME" to the complete path of the top level Wired directory, such as "c:\WiredBaBar". There should be no colon or semicolon after this directory name.
    • NT users who have administrative privileges on their machines can make this WIRED_HOME setting permanent by going to the
      • Start Menu...Settings...Control Panel...System...Environment
      • Add a new Variable, "WIRED_HOME" (careful, make it all Caps)
      • Set the value to something like
      • C:\WiredBaBar
      • Then be sure to BOTH hit the Apply button and then the OK button.
      • All new command windows will then have these settings (old ones will not get the new settings).
      • To review the current settings in any command window, just type "set"
  • Special for Unix or Linux:
    • Set the environment variable "WIRED_HOME" to the complete path of the top level Wired directory, such as "c:/WiredBaBar". There should be no colon or semicolon after this directory name.
    • Set the executable bits on for every directory in the application by going to the WiredBaBar directory and typing: 
         find . -type d -exec chmod +x {} \;

To Upgrade to a Newer Copy of the BaBar WIRED Client Application

The client application is entirely self-contained; it does not add or modify any files outside of its own file structure. The upgrade procedure is therefore very simple.
  • Delete the entire file structure of the old client copy (if you have taken the defaults, this will be the structure that begins with WiredBaBar).
  • Download a new client copy.
  • Rerun the install procedure as described above.
  • If you do the new installation to the same directory that had the old installation, there will be no need to redo any environment variable settings.

To Run the BaBar WIRED Client Application

Starting WIRED

From the WiredBaBar/bin directory, start wired by typing the command: 
   WiredBaBar/bin> wired
  • Special for SLAC Unix or Linux:
    • If you have the standard BaBar setup, you can just type "wired" from any directory.
  • Special for Macintosh:
    • Just click on one of the application icons created by the installation procedure.
    • Note that for now only the CES_ANALYSIS and CES_BETA startup procedures will work as advertised, but once these applications are started you can change to any other server through the event chooser.
  • Special for NT:
    • If you add the WiredBaBar/bin directory to your PATH (following the same procedure as was described earlier to set CLASSPATH), you can also start Wired by selecting "wired.bat" from the Start menu's "Wired BaBar" program group.
