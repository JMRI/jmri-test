Installing Java on various plaforms

This page discusses how to install Java on various platforms. Please note that you might not need to do this! Many computers now come with Java installed, or have it installed during the normal software update process.

Installing Java on Macintosh Classic

Macintosh users should get Java version MRJ 2.4, which corresponds to Java 1.1.8, from Apple at http://devworld.apple.com/java/text/download.html

Installing Java on MacOS X

MacOS X comes with a working Java, which can be operated much like the Linux form. It shouldn't be necessary to install anything.

Installing Java on Linux

Start by making sure you have an appropriate version of Java on your system, a version number of 1.2 or greater. You can check the version number by typing 
   java -version

If you did not install Java on your system yourself, be aware that some systems have Java only partially installed. They will respond correctly to the above command, but still are not all there. If your system does not have Java in the environment variable PATH, the install is probably not complete. Check with the system manager or do a fresh Java installation.

You can obtain a Linux version of Java free from Sun at http://java.sun.com/j2se

The same page contains links to third party versions of Java for other platforms.

If you are using Java 1.1, Set the environment variable CLASSPATH as follows (being sure to use all capitals).
The exact setting of CLASSPATH depends on where you placed the Java code, but it should start with "." to include the current directory, and it should then specify where the basic Java classes are to be found. Examples are:

setenv CLASSPATH ".:/usr/lib/jdk1.1/lib/classes.zip"

Installing Java on Windows

Start by making sure you have an appropriate version of Java on your system, a version number of 1.2 or greater. You can check the version number by typing 
   java -version

If you did not install Java on your system yourself, be aware that some systems have Java only partially installed. They will respond correctly to the above command, but still are not all there. If your system does not have Java in the environment variable PATH, the install is probably not complete. Check with the system manager or do a fresh Java installation.

You can obtain the Windows version of Java free from Sun at http://java.sun.com/j2se

The same page contains links to third party versions of Java for other platforms.

If you are using Java 1.1, Set the environment variable CLASSPATH as follows (being sure to use all capitals).
The exact setting of CLASSPATH depends on where you placed the Java code, but it should start with "." to include the current directory, and it should then specify where the basic Java classes are to be found. Examples are:

set CLASSPATH=.;C:\jdk1.2\lib\classes.zip;

  • NT users who have administrative privileges on their machines can make this CLASSPATH setting permanent by going to the
    • Start Menu...Settings...Control Panel...System...Environment
    • Add a new Variable, "CLASSPATH" (careful, make it all Caps)
    • Set the value as indicated above
    • Then be sure to BOTH hit the Apply button and then the OK button.
    • All new command windows will then have these settings (old ones will not get the new settings).
    • To review the current settings in any command window, just type "set"
    • In some cases, the new settings do not take effect until after a reboot.
  • Windows 9x users can make the CLASSPATH setting permanent by doing the CLASSPATH equivalent of the procedure for making PATH changes permanent that you will find at http://java.sun.com/products/jdk/1.2/install-windows.html#Environment
  • Windows 9x users should also:
    set OS=W98 (or 95, etc)
    increase environment space as explained at: http://support.microsoft.com/support/kb/articles/Q230/2/05.ASP
