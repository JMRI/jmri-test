JMRI: Site Web

Cette page présente les aspects techniques du site web JMRI.

Si vous voulez juste savoir comment faire un petit changement dans un site Web ou la page d'aide, s'il vous plaît voir la page séparée des instructions .

Structure de l'Information

Nous distinguons trois types d'informations sur le site Web principal:
  1. Les informations pour l'usager qui sont utile au moment de l'exécution. Ceci est rendu disponible à la fois via notre JavaHelp , et aussi directement en plaçant les fichiers sur le Web.
  2. Les Informations pour l'usager qui sont soit inutile au moment de l'exécution, tels que instructions pour l'installation du logiciel, ou sont trop grande pour l'inclusion dans les version, par exemple, tutoriels vidéo.
  3. Les informations de référence à partir de versions, telles que les définitions décodeur, des copies de scripts, JavaDocs, etc Nous fournissons ces informations séparément:
    1. L'utilisateur Run-time est disponible en plaçant le système d'aide sur le site web.
    2. Les informations d'autres utilisateurs sont stockées dans le module CVS du site.new , et placées directement sur le site web.
    3. Les informations de références sont mises sur le site web de leurs propres modules CVS, ou créés par des scripts Ant automatisés au cours de la constructions des versions.

Technologie

La mise en page se fait en utilisant CSS, initialement créé par John Plocher. Toutes les pages doivent référencer les fichiers CSS pour l'écran et l'impression du répertoire "/css" . Cela signifie également que vous devez quitter le formatage des feuilles de style, et de minimiser le formatage explicite que vous faites directement dans HTML.

Parce que nous utilisons nos pages Web dans JavaHelp, il y a des restrictions sur l'utilisation des étiquettes. Voir nos pages web JavaHelp pour plus de détails.

Nous nous dirigeons vers l'aide serveur -Side Includes pour fournir une information en-têtes, des encadrés et des pieds de page. Cela permettra également de nous faire part du contenu HTML entre le web et le système JavaHelp utilisé par le programme lui-même. Chaque page ne contiendra que le contenu, en plus comprend les fichiers nommés "tête", "Sidebar" et "footer" qsi ui contiennent les informations de navigation. Finalement, toutes les pages d'index, mais peu ont l'extension ". shtml". (Les autres pages index.html gardent ce nom afin que les gens qui demandent juste une URL répertoire leur sera servi quelque chose d'utile)

Site Web Local: Mac OS X

Si vous voulez héberger une copie du site web JMRI sur votre ordinateur local Mac OS X , suivez ces instructions d'abord configurer le serveur Apache pour faire à côté un serveur inclut, puis faire une copie fichiers disponible du site web sur le serveur.
  1. Vous devez modifier le fichier de configuration du serveur web Apache.
    • Sur Mac OS X 10.4 (Tiger), c'est le fichier/etc/httpd/httpd.conf
    • Sur Mac OS X 10.5 (Leopard), c'est le fichier etc/apache2/httpd.con /
    Une fois que vous avez localisé le fichier, ouvrez le avec votre éditeur favori (Vous devez vous authentifier avec le mot de passe administrateur pour modifier ce fichier).
    • Pour activer les inclusions sur le serveur, trouver la section qui ressemble à 
      # Ceci peut également être «Aucun», «Tous», ou toute combinaison de «indices»,
# "Includes", "FollowSymLinks", "ExecCGI", ou "MultiViews".
#
# Notez que "MultiViews" doit être nommé explicitement * * --- "Options Tous"
# si si si si ne vous la donne pas.
#
Options Indexes FollowSymLinks MultiViews
      Ajouter le mot «Includes» à la dernière ligne de sorte qu'il ressemble 
          Options Includes FollowSymLinks Indexes MultiViews
    • Maintenant, configurez le serveur pour gérer les fichiers avec les extensions ". shtml" . Trouvez le section qui ressemble à: 
           #
     # Pour utiliser les fichiers du serveur analysé HTML 
     #
     # AddType text / html. shtml
     Shtml # AddHandler server-parsed.
      et décommenter les deux dernières lignes de sorte qu'il ressemble à 
           #
     # Pour utiliser  les fichiers du serveur analysé HTML
     #
     AddType text / html. shtml
     Shtml AddHandler server-parsed.
  2. Pour consulter une copie des fichiers:
    • Changer le répertoire du serveur web 
      cd /Library/WebServer/Documents
    • Consultez les fiches correspondantes
      cvs co site.new
mv site.new/* .
mv site.new/.cvsignore .
rmdir site.new
cvs co help
cvs co jython
cvs co resources
cvs co xml
      La commande "mv" déplacer les fichiers à partir des modules CVS "site.new" au niveau de la racine de l'espace fichier serveurs web, de sorte qu' ils sont accessibles comme de simples URL .
  3. Redémarrez votre ordinateur pour réinitialiser le serveur web.
  4. Testez en essayant d'afficher l'URL http://localhost/ , qui devrait vous une page d'index JMRI.
