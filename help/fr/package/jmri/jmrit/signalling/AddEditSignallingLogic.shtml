Ajout/Edition Logiques de Mât de Signal

La première chose que vous devez faire lorsque vous créez une Logique Mât de Signal est de sélectionner les mâts de Signalisation source suivi par ceux de destination. Le mât du signal source est un dont vous souhaitez contrôler l'ensemble de ses apparences. Le mât de signal de destination est le mât de signal suivant sur la ligne , l'apparition du mât de destination ainsi que l'état des aiguillages et des cantons auront un effet direct sur la l'apparence du mât du signal source.
Si vous n'utilisez pas l'éditeur de Réseau, alors AUCUNE validation n'est effectuée sur les mâts source et de destination. Si l'éditeur réseau est utilisé, alors le système va essayer de déterminer si le mât de destination est accessible à partir du mât source.

En Mode Édition Uniquement La vitesse maximale admissible entre le chemin séparant les deux mâts de signaux est affiché, cela est déterminé par la vitesse configurée en raison des cantons et les aiguillages.

La case "Utiliser Chemins Éditeur de Réseau" vous permet d'obtenir le système pour générer automatiquement les cantons nécessaires, l'aiguillage et le mâts de signaux avec leurs états qui sont situés entre la source et le mâts de signaux de destination.
En sélectionnant "Utilisation de Chemins Éditeur de Réseau", deux cases à cocher supplémentaires apparaissent, "Utiliser Détails Aiguillage de l'Éditeur de Réseaut » et «Utiliser les Détails des Cantons de l'Éditeur de Réseau" Sélection une de ces informations permettra de déterminer ce qui de l'éditeur de réseau est utiliséelorsque le système génère automatiquement l'itinéraire.
Les informations générées automatiquement ne sont pas enregistrées avec le fichier du panneau, cet information est générée chaque fois que le panneau de disposition est ouvert.
Si les informations sur le panneau éditeur de réseau sont éditées, le changement n'est pas reflété dans la Logique SignalMast (Mât de Signaux), vous aurez besoin d'enregistrer le panneau, fermer JMRI et re-ouvrir le panneau pour la logique soit mise à jour.

L'option "Autoriser La logique à déterminer automatiquement les conflits SignalMast" tente de déterminer quels signaux pourraient causer un itinéraire incompatible s'ils n'étaient pas mis sur danger.

La fonctionnalité "verrouillage des aiguillages lorsque la Logique mât de signaux est actif" vous permet de désactiver de manière sélective les commandes d'accessoire (aiguillages) à partir de cab (régulateur) et/oude boutons-poussoirs locaux câblés. JMRI désactive les poussoirs en envoyant une commande désactiver les bouton-poussoirs aux décodeurs d'aiguillages lorsque l'itinéraire est bloqué, , et permet les boutons poussoirs lorsque l'itinéraire est déverrouillé. Lorsque le verrouillage d'un aiguillage à partir de commandes cab, le programme JMRI suivra l'adresse accessoire de l'aiguillage et si elle détecte une commande de changement d'un cab, va annuler la commande immédiatement en envoyant la commande opposé à l'aiguillage. Tous les aiguillages qui sont surveillés par le programme JMRI et sont inclus dans l'itinéraire seront verrouillés.

Le bouton radio en dessous du "Aiguillage Verrouillé quand Mât de Signal Actif" vous permet de n'afficher que les aiguillages, capteurs, cantons ou des mâts de signaux qui font partie de cette logique mât de signal.

Il y a quatre onglets dessous le bouton "Montrer", chacun contient les détails de cantons, aiguillages, des capteurs et signalmasts qui composent la logique Signalmast. Dans chacun de ces onglets, il y a une table modifiable qui vous permet de sélectionner manuellement l'élément qui doit être inclu dans la logique de mât de signal et de l'Etat dans lequel cet élément devrait être positionné
Ci-dessous est un autre tableau qui montre les éléments automatiquement générés (sauf dans l'onglet du capteur). Un éléments saisit manuellement l'emportera sur l'automatique éléments générés, donc si vous souhaitez l'emporter sur n'importe quel élément automatique,listez les dans une table manuelle sélectionnez l'élément correspondant et définissez son état. Cliquez sur l'option "Inclure" dans l'une des tables pour ajouter l'élément désiré à votre logique signalmast. A côté de l'option "Inclure" se trouve l'option d'état dans lequel l'élément doit être pour que le Mat de Signal devienne Actif Par exemple, si vous souhaitez que l'aiguillage soit dévié pour déclencher la logique, alors sélectionnez "déviés" dans le menu déroulant. Si vous souhaitez l'emporter sur un aiguillage qui a été automatiquement inclus, et vous ne vous souciez de l'état dans lequel il est réglé, sélectionnez «Tout Etat» à partir du menu déroulant.

The radio button below the "Lock Turnout when Signal Mast Logic is Active" allows you to show only the turnouts, sensors, blocks or signal masts that are part of this signal mast logic.

There four tabs below the "Show" button, each contains the details of Blocks, turnouts, sensors and signalmasts that make up the Signalmast logic. In each of these tab, there is an editable table that allows you to manually select which item should be included in the Signal Mast logic and the state which this item should be in.
Below that is a further table that shows the automatically generated items (except in the sensor tab). Manually entered items over-ride the automatically generated items, therefore if you wish to over-ride any item in automatically generated list then in the manual table select the corresponding item and set the state. Click the "Include" box in any of the tables to add the desired item to your signalmast logic. Next to the "Include" box is the state option which that item must be in for the Signal Mast to come active. For example, if you wish the turnout to be in a thrown state to trigger the logic, then select "Thrown" in the drop down box. If you wish to over-ride a turnout that has automatically included, and you don't care what state it is set to, select "Any State" from the drop down box.

Update Signal Mast Logic

This will add or update your Signal Mast Logic to the system, once added, the tables holding the automatic generated data will then be re-populated.
