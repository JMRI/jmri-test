JMRI: Aspect Signaling

Controling signaling in a prototypical manner

Protoypical signal systems present "aspects" to the engineer to tell him how to run his train. Different signals may show an aspect using different combinations of lights, semaphore positions, and/or placards; these are called "appearances". For example, the "stop" aspect with be a single red light on a signal mast with one lamp, red over red lamps for a signal mast with two lamps, etc.

Available Signaling Systems

The available signal system definitions are found in the xml/signals directory.
basic
A non-prototypical set of aspects and appearances that most model railroaders will understand easily.
AAR-1948
American Association of Railroads 1948 rule book
SPTCO-1960
Southern Pacific Railway and Transportation Company 1960 rule book

Creating a New Signaling System

For now, you need to manually create a new directory under the "xml/signals" directory to hold your new signal definition. Then, provide these files:
  • index.shtml - Free-form decription of the signal system
  • aspects.xml - Define the complete set of available aspects
  • appearance-*.xml - One file for each type of SignalMast, defining how to display each aspect.
