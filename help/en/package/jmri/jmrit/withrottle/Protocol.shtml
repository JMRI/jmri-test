JMRI: WiFi Throttle Communications Protocol

This page describes the protocol provided by the jmri.jmrit.withrottle package for controlling trains through JMRI via a TCP/IP link.

Additions and corrections are welcomed. This information is very preliminary.

Open a TCP/IP link to the port advertised via ZeroConf/Bonjour as the servicve "_withrottle._tcp.local.".

Send TLaddr, CTLaddr, TSaddr, or CTSaddr to allocate a throttle. "L" means Long DCC address, "S" means Short DCC address.

TR0 is reverse, TR1 is forward.

TViiii sets speed, where iiii is an integer value. (Need to figure out range for this)

Tr and Td "release" and "dispatch" the throttle. For most DCC systems, these are the same thing; if in doubt, use release.
