JMRI: showPanel web application

Background

The JMRI FrameServlet is a useful tool for extending JMRI to browsers by periodically grabbing a screen image of a requested window (running on a JMRI server), displaying it in the browser, and transmitting any mouse clicks back to the JMRI window. Unfortunately, there are some drawbacks to this approach (delayed response, heavy server load, no highlighting of active elements, mouse "sharing", etc.).

With the addition of the JMRI PanelServlet, it became possible for a client app to retrieve complete details of a panel, enough to recreate the panel in the client. The XMLIOServlet provides for updating JMRI elements, and for monitoring changes to those elements.

New web application

showPanel.html is a modern web application which draws active JMRI panels on a browser screen. It retrieves panel definitions from JMRI via the PanelServlet and builds panel client-side, including click functions. It sends changes and listens for changes to panel elements using the xmlio server.

Links to all available panels are shown on the JMRI Web Server home page (you might need to rebuild it in Prefs to see the links)

The base url is "/web/showPanel.html". With no parms, the page will list links to available panels, or you can pass a parm of the format "?name=[type]/[panelname]" to open the specified panel directly. For example: http://192.168.0.7:12080/web/showPanel.html?name=Layout/MyPanel"

This app requires a modern "HTML5" browser (Chrome, FireFox, Safari, IE9).
