JMRI: Obtention du Code

Mise en route

Vous pouvez télécharger le code source le plus récent via SVN en utilisant: 
svn checkout svn://svn.code.sf.net/p/jmri/code/trunk/jmri

Détails

Notre code source est distribué via SVN. (Nous gardons aussi le site web dans le SVN, si vous souhaitez travailler directement avec lui.) Si vous voulez travailler avec les code, cela vaut la peine de prendre quelques minutes pour installer SVN . Nous sommes désolés, mais nous ne pouvons pas appuyer d'autres méthodes (envoi des fichiers. zip, etc), car ils sont tout simplement sources de trop d'erreurs.

Pour obtenir une copie de la source pour votre propre usage, vous pouvez utiliser "checkout anonyme". Pour un client en ligne de commande cvs, créez un nouveau répertoire pour le projet, changer de répertoire actif, et dire que vous existez à SVN 

svn checkout svn://svn.code.sf.net/p/jmri/code/trunk/jmri
Ce "login " en ligne n'est nécessaire qu'une seule fois, après quoi, il est rappelé sur votre machine locale. Vous pouvez être sollicité pour un mot de passe. Si oui, appuyez simplement sur le retour, sinon le mot de passe vide ne sera acceptée. Vous pouvez alors obtenir une copie du code de 
svn checkout svn://svn.code.sf.net/p/jmri/code/trunk/jmri

Si vous utilisez un client graphique de base comme WinCVS, déroulez le menu "admin" et sélectionnez "ligne de commande" pour obtenir une place pour taper ces commandes. Sur d'autres ordinateurs, vous pouvez taper directement sur la ligne de commande.

Si vous avez une copie existante du code et que vous souhaitez mettre à jour pour le contenu le plus récent de SVN, la deuxième commande est la suivante: 

cvs-d: pserver: anonymous@jmri.cvs.sourceforge.net: / cvsroot / jmri-q update-dA

Voir aussi la page FAQ SVN .

Après que l'extraction soit terminée, vous trouverez des répertoires appelés "xml", "ressources", "jython", "help ", "lib", "java" et d'autres. Modification du répertoire "java". Là, vous trouverez un répertoire "src" qui contient le code source de JMRI lui-même, et un répertoire "tests" qui contient les routines de test différentes.

Vous pouvez regarder le code avec votre éditeur de texte favori. Pour compiler et l'exécuter, vous avez besoin de l'un des outils de compilation qui est décrit sur la barre à à gauche: Ant, JBuilder, etc (Ant peut être plus facile à installer et à utiliser)

Une fois que vous travaillez avec le code, vous aurez probablement créer des améliorations et vous souhaitez contribuer à la vie du projet. C'est là que SVN a énormément d'avantages, avec les autorisations appropriées et les commandes, vous pouvez fusionner vos modifications dans la source commune automagiquement. Contactez-nous si vous êtes prêt à le faire ou voir les infos sur la contribution .

Détails Windows

Obtenir et mettre à jour le code avec TortoiseCVS

Pour obtenir une vérification initiale de CVS pour votre propre usage, après l'installation http://www.tortoisecvs.org/. Créez un nouveau répertoire pour le projet, par exemple créer un dossier nommé "JMRISourceCode". Clic droit (control-clic) sur ce dossier et sélectionnez CVS CVS Checkout ". Dans le volet qui s'ouvre pour le CVSROOT entrez <: pserver: jmri.cvs.sourceforge.net: / cvsroot / jmri> Pour le serveur, entrez . En bas à droite de l'écran est un "Récupérer la liste" Cliquez dessus et une liste s'affiche, cliquez sur le «all». Cliquez ensuite sur le bouton OK au bas du volet. Un nouveau volet s'ouvre et les fichiers seront téléchargés, sur le bas de ce volet est une case à cocher appelée "Close on Completion", la cocher. Quand il se ferme, le dossier que vous aurez mis dans le conteneur dossiers aura tous les éléments JMRI.

Une fois que vous avez effectué l'extraction initiale de CVS dans le dossier, pour les mettre à jour les sélectionner tous , puis clic droit et sélectionnez «cvs update ». Vous pouvez aussi simplement mettre à jour un seul dossier en le sélectionnant.
